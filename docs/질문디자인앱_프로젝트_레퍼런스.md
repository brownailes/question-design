## 2. 기술 스택 및 의존성

### 프론트엔드 (클라이언트 전용)
- **HTML/CSS/JS** — 단일 파일, 프레임워크 없음
- **폰트**: Google Fonts (`Inter`, `JetBrains Mono`)
- **디자인 시스템**: Apple 스타일 미니멀리즘, CSS Custom Properties

### AI / API
- **Gemini API**: `v1beta` 엔드포인트, `generateContent` 메서드
- **응답 형식**: `responseMimeType: "application/json"` (구조화된 JSON 응답)
- **토큰 설정**: `maxOutputTokens: 4096`, `temperature: 0.8`, `topP: 0.95`

### 배포
- **호스팅**: GitHub Pages (master 브랜치)
- **DNS**: 가비아(Gabia) — `question` CNAME → `brownailes.github.io.`
- **HTTPS**: GitHub Pages 자동 인증서

### 피드백 수집
- **Google Apps Script** → Google Sheets 연동

## 3. 핵심 기술적 결정 및 교훈

### 🔑 API 모델 진화 (가장 중요한 교훈)

> [!warning] API 모델은 빠르게 deprecated 된다
> Google Gemini의 모델 라이프사이클은 매우 짧다. 코드에 모델명을 하드코딩하면 수개월 내 작동이 멈출 수 있다.

| 시점 | 모델 | 상태 |
| :--- | :--- | :--- |
| 초기 개발 | `gemini-1.5-flash` | ❌ deprecated (v1beta에서 404) |
| 1차 수정 | `gemini-2.0-flash` | ❌ "no longer available to new users" |
| **최종 (현재)** | **`gemini-2.5-flash`** | ✅ 정상 작동 |

**교훈**: 향후 모델 변경 시, `ListModels` API를 먼저 호출하여 사용 가능한 모델을 확인한다.
```
curl -s "https://generativelanguage.googleapis.com/v1beta/models?key=$KEY" | grep "name"
```

### 💰 무료 티어 vs 유료 빌링

> [!important] 무료 티어의 한계
> 무료 API 키는 일일 요청 한도(1,500 RPD)가 있으며, 한도 초과 시 `429 Too Many Requests` 에러가 발생한다. 이 한도는 **하루 단위**로 리셋되므로, 분당 제한과 혼동하면 안 된다.

- **무료 티어**: 1,500 RPD, 15 RPM, IP당 제한
- **유료 빌링**: 무제한 RPD, 2,000 RPM
- **비용**: Gemini 2.5-flash 기준, 질문 1회 생성당 약 ₩0.1 미만 (사실상 무료)
- **설정 방법**: [Google AI Studio](https://aistudio.google.com/apikey)에서 프로젝트에 빌링 활성화

### 🔐 보안 — API 키 관리

> [!caution] 서버리스 환경에서의 키 보호 한계
> 정적 HTML에 포함된 API 키는 브라우저 개발자 도구로 누구나 확인할 수 있다. 난독화(Obfuscation)는 자동 스캐너를 우회할 수 있을 뿐, 사람이 직접 들여다보면 키를 추출할 수 있다.

**현재 방식 (난독화)**:
```javascript
const _K = ['AIzaSy', 'BIPrnL', '5_iRUy', 'f-NDq1', 'gYz1lU', 'yu0Z10', 'sac'].join('');
```
- **장점**: GitHub의 자동 키 탐지 봇에 걸리지 않아 키가 자동 취소되지 않음
- **한계**: 코드를 읽는 사람이면 누구나 키 복원 가능

**향후 보안 강화 방안**:
- Google Cloud Console에서 **API 키 제한** 설정 (HTTP Referrer 제한: `question.juailes.com/*`)
- 일일 사용량 상한(Quota limit) 설정으로 악의적 대량 호출 방지
- 장기적으로는 **백엔드 서버**(Supabase Edge Functions 등)로 키를 완전히 숨기는 방식 고려

### 🧠 프롬프트(지식) 보호

> [!note] 서버리스의 근본적 한계
> `buildSystemPrompt()` 함수 안에 있는 질문 설계 원리, 교육과정 매핑, 4단계 질문 구조 등은 현재 누구나 소스 보기로 확인할 수 있다. 이 프롬프트가 핵심 지적 재산이라면, 백엔드 서버 도입이 필수적이다.

- **현재**: 오픈소스처럼 공개된 상태
- **향후**: Google Apps Script 또는 Supabase/Cloudflare Workers 등을 활용하여 프롬프트를 서버 측으로 이전

### 📦 JSON 파싱 안정성

> [!tip] Gemini 응답은 항상 "불완전"할 수 있다
> LLM의 출력은 토큰 한도, 네트워크 타임아웃 등으로 인해 JSON이 중간에 잘릴 수 있다.

**적용된 방어 로직**:
1. 마크다운 코드 펜스(` ```json ``` `) 자동 제거
2. 잘린 JSON 자동 복구 (누락된 `]`, `}` 보완)
3. 불완전 문자열(unterminated string) 제거
4. `maxOutputTokens`를 넉넉하게 설정 (4096)

## 4. UI/UX 설계 원칙

### 디자인 가이드라인
- **Apple 미니멀리즘**: 여백 중시, 정보 위계 명확, 둥근 모서리(16px radius)
- **타이포그래피 4계층**: Overline(배지) → H1(헤드라인) → Tagline(세미볼드) → Subtitle(연한 보조)
- **컬러 시스템**: CSS Custom Properties로 일관된 색상 관리

### 주요 UX 결정

| 결정 | 이유 |
| :--- | :--- |
| 학년군 우선 선택 → 교과 필터링 | 교과 목록이 너무 길어 사용자 혼란 방지 |
| 커스텀 상향 드롭다운 | 네이티브 select의 하단 확장은 스크롤 필요 → 위로 열어 전체 목록 가시성 확보 |
| 토스 스타일 UX Writing | 에러·안내 메시지를 친절하고 명확하게 → "학년군과 교과를 선택하면 질문을 생성할 수 있어요" |
| 결과 카드에 좌측 컬러 보더 | 4단계 질문의 시각적 구분감 강화 (빨강·주황·파랑·노랑) |
| 교육적 의도를 별도 박스로 분리 | 질문과 의도를 명확히 구분하여 전문적 인상 부여 |

## 5. 배포 워크플로

### GitHub Pages 배포 절차
```bash
cd /tmp/question-design
git add index.html CNAME
git commit -m "커밋 메시지"
git push origin master
```

### 커스텀 도메인 설정 (가비아)
1. 가비아 DNS 관리 (`dns.gabia.com`) 로그인
2. `juailes.com` → DNS 설정 → 레코드 수정
3. CNAME 레코드 추가: `question` → `brownailes.github.io.`
4. GitHub 저장소에 `CNAME` 파일 추가 (내용: `question.juailes.com`)
5. DNS 전파 대기 (약 10~30분)

### 캐시 주의사항

> [!warning] 브라우저 캐시
> GitHub Pages는 CDN 캐시를 사용한다. 배포 후 즉시 반영되지 않을 수 있으므로, 테스트 시 반드시 **강력 새로고침**(`Cmd+Shift+R`)을 수행한다.

## 6. 데이터 구조

### 교육과정 매핑 (`CURRICULUM` 객체)
- 학년군: 초등 1~2, 초등 3~4, 초등 5~6, 중학교, 고등학교
- 교과: 학년군별로 다른 교과 목록 (국어, 수학, 사회, 과학 등)
- 기본값: "전체" 선택 시 모든 교과 표시

## 8. 트러블슈팅 가이드

| 증상 | 원인 | 해결 |
| :--- | :--- | :--- |
| "models/gemini-X.X is not found" | 모델 deprecated | `ListModels` API로 사용 가능 모델 확인 후 코드 업데이트 |
| "Quota exceeded" (limit: 0) | 일일 무료 한도 소진 | 빌링 활성화 또는 새 API 키 발급 (다른 Google 계정) |
| "동시 접속자가 많아요" (429) | 분당 요청 한도 초과 | 1분 대기 후 재시도 (자동 안내 메시지 표시됨) |
| "Unterminated string in JSON" | 응답 토큰 한도 부족 | `maxOutputTokens` 증가 (현재 4096) |
| 배포 후 변경 미반영 | CDN/브라우저 캐시 | `Cmd+Shift+R` 강력 새로고침 |
| 사이트 연결 불가 | CNAME/DNS 미설정 | 가비아 DNS에 CNAME 추가 + GitHub CNAME 파일 확인 |
