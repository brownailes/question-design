# 🚀 질문 디자인 앱 개발 일지 — 2026-03-27

## 🛠️ 기술적 개선 사항

### 1. LLM 모델 업데이트 (`gemini-1.5-flash` → `gemini-2.0-flash`)
- **이유**: 기존 모델이 deprecated되어 API 호출 오류(404/400) 발생.
- **조치**: 최신 `gemini-2.0-flash` 모델로 코드를 업데이트하여 안정적인 질문 생성 환경 구축.

### 2. 학년군 우선 선택 로직 도입
- **변경 전**: 학년군과 교과를 동시에 임의로 선택.
- **변경 후**: **학년군(Grade Group)**을 먼저 선택해야 하며, 선택된 학년군에 따라 **교과(Subject)** 목록이 동적 필터링됨.
- **기본값**: '전체'로 설정하여 처음에는 모든 교과가 보이게 하여 선택 편의를 도모함.

### 3. 커스텀 상향 드롭다운 (Custom Custom Select Component)
- **이유**: 교과 목록이 길어지면 하단으로 열리는 네이티브 select는 스크롤을 내려야 전체를 볼 수 있음 (사용자 불편).
- **조치**: CSS/JS로 커스텀 드롭다운을 제작. `bottom: calc(100% + 6px)` 설정을 통해 **위쪽으로** 열리게 하여, 입력폼 위에서 전체 목록을 한눈에 확인할 수 있도록 함.

### 4. UX Writing 및 피드백 강화 (Toss 스타일)
- **메시지 개선**: 
  - 비활성 버튼 클릭 시: "학년군과 교과를 선택하면 질문을 생성할 수 있어요." (Pill 형태 토스트)
  - 할당량 초과(429 Error) 시: "지금 동시 접속자가 많아요. 1분 뒤에 다시 시도해 주세요."
- **목표**: 사용자가 에러 상황에서도 당황하지 않고 다음에 무엇을 해야 할지 명확히 알 수 있게 함.

### 5. 배포 최적화 (GitHub Pages)
- **이력**: CNAME(`question.juailes.com`) 파일 추가 및 DNS 이슈 발생.
- **조치**: 즉각적인 공유를 위해 CNAME을 제거하고 GitHub 기본 URL(`https://brownailes.github.io/question-design/`)로 배포 완료. HTTPS 강제 옵션 활성화.

## 📝 교육 및 기획적 교훈

- **프롬프트 노출**: 서버리스(HTML 단일 파일) 방식은 배포가 매우 빠르다는 장점이 있지만, 소스코드 내에 심어둔 '질문 설계 원리(System Prompt)'가 외부에 노출될 수 있음.
- **보안/IP 보호**: 향후 본격적인 서비스화 시에는 백엔드(Server) 연동을 통해 프롬프트를 보이지 않게 처리해야 할 필요성을 확인.
